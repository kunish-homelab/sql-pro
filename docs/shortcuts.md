# Keyboard Shortcuts

SQL Pro is designed for keyboard-driven productivity. This reference documents all available keyboard shortcuts organized by category.

## Quick Reference

| Action              | macOS                             | Windows/Linux                      |
| ------------------- | --------------------------------- | ---------------------------------- |
| Command Palette     | <kbd>Cmd</kbd> + <kbd>K</kbd>     | <kbd>Ctrl</kbd> + <kbd>K</kbd>     |
| Execute Query       | <kbd>Cmd</kbd> + <kbd>Enter</kbd> | <kbd>Ctrl</kbd> + <kbd>Enter</kbd> |
| Open Settings       | <kbd>Cmd</kbd> + <kbd>,</kbd>     | <kbd>Ctrl</kbd> + <kbd>,</kbd>     |
| Open Database       | <kbd>Cmd</kbd> + <kbd>O</kbd>     | <kbd>Ctrl</kbd> + <kbd>O</kbd>     |

## Vim Mode

SQL Pro supports optional Vim-style keybindings for power users. There are two separate Vim modes:

1. **Editor Vim Mode** - Vim keybindings in the SQL editor
2. **App Vim Mode** - Vim-style navigation throughout the app (sidebar, data grid)

Enable these in **Settings** > **Vim Settings**.

### Editor Vim Mode

When enabled, the SQL editor supports standard Vim keybindings:

| Mode   | Key                                                 | Action              |
| ------ | --------------------------------------------------- | ------------------- |
| Normal | <kbd>h</kbd> <kbd>j</kbd> <kbd>k</kbd> <kbd>l</kbd> | Move cursor         |
| Normal | <kbd>w</kbd> <kbd>b</kbd> <kbd>e</kbd>              | Word navigation     |
| Normal | <kbd>i</kbd>                                        | Enter insert mode   |
| Normal | <kbd>a</kbd>                                        | Append after cursor |
| Normal | <kbd>o</kbd>                                        | Open line below     |
| Normal | <kbd>O</kbd>                                        | Open line above     |
| Normal | <kbd>d</kbd><kbd>d</kbd>                            | Delete line         |
| Normal | <kbd>y</kbd><kbd>y</kbd>                            | Yank (copy) line    |
| Normal | <kbd>p</kbd>                                        | Paste               |
| Normal | <kbd>/</kbd>                                        | Search              |
| Insert | <kbd>Escape</kbd>                                   | Exit to normal mode |

::: tip Vim Status Bar
When Editor Vim Mode is enabled, a status bar appears below the editor showing the current mode (NORMAL, INSERT, VISUAL, etc.) and pending commands.
:::

### App Vim Mode (Sidebar & Data Grid)

When App Vim Mode is enabled, you can navigate the sidebar and data grid with Vim keys.

#### Navigation

| Key                      | Action               |
| ------------------------ | -------------------- |

| <kbd>$</kbd>             | Jump to last column  |

#### Actions

| ----------------------------------- | ------------------------------------------------ |

## Tips for Keyboard-Driven Workflow

### Maximize Productivity

1. **Learn the Command Palette** (<kbd>Cmd/Ctrl</kbd> + <kbd>K</kbd>) - It's the fastest way to access any command
2. **Use Tab navigation** in data grids for fast horizontal data entry
3. **Enable Vim Mode** if you're familiar with Vim - it significantly speeds up navigation
4. **Memorize query execution** (<kbd>Cmd/Ctrl</kbd> + <kbd>Enter</kbd>) - You'll use it constantly

### Efficient Query Writing

1. Press <kbd>Cmd/Ctrl</kbd> + <kbd>2</kbd> to jump to the query editor
2. Write your SQL with autocomplete (triggered automatically)
3. Press <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>F</kbd> to format
4. Press <kbd>Cmd/Ctrl</kbd> + <kbd>Enter</kbd> to execute
5. Press <kbd>Cmd/Ctrl</kbd> + <kbd>H</kbd> to review history

### Efficient Data Editing

1. Navigate to a cell using arrow keys or Tab
2. Press <kbd>Enter</kbd> to start editing
3. Make your change
4. Press <kbd>Tab</kbd> to save and move to the next cell
5. Press <kbd>Cmd/Ctrl</kbd> + <kbd>Shift</kbd> + <kbd>S</kbd> to review changes
6. Apply changes from the diff preview panel

## Troubleshooting

### Shortcuts Not Working

If keyboard shortcuts aren't responding:

1. **Check focus** - Make sure the correct panel is focused
2. **Check for conflicts** - Browser or OS shortcuts may override app shortcuts
3. **Check Vim mode** - Some shortcuts behave differently in Vim mode
4. **Refresh the page** - Try reloading if shortcuts become unresponsive

### Browser Conflicts

Some shortcuts may conflict with browser defaults:

- <kbd>Cmd/Ctrl</kbd> + <kbd>H</kbd> might open browser history in some browsers
- <kbd>Cmd/Ctrl</kbd> + <kbd>1</kbd>/<kbd>2</kbd> might switch browser tabs

SQL Pro attempts to prevent these defaults when focused, but if you experience conflicts, try using the Command Palette instead.
